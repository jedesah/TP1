LOG1000 - Travail pratique #1 - Construction logiciel et gestion de révision

Objectifs

  • Comprendre l'utilité et le fonctionnement des systèmes de gestion de révision
  • Développer une compétence de base avec Git, un logiciel de gestion de révision communément utilisé pour les logiciels à source libre
  • Développer une compétence de base avec le système d’exploitation Linux pour ceux qui n’y sont pas déjà familiés

Éxigences

  • Compléter le tutoriel suivant seul ou en équipe de 2
  • Remettre sur Moodle votre entrepôt Git sous la forme d'une archive
  • Remettre sur Moodle votre fichier patch de votre changement à V8

Critère de correction

  • Conformité de votre entrepôt Git à ce qui est attendu: 80%
  • Conformité de votre fichier patch pour V8: 20%

Tutoriel 1

Introduction

Il s'est produit, il n'y a pas si longtemps, une catastrophe de nature apocalyptique qui a replongé la société humaine à l'âge noir du savoir. Il reste encore certains artéfact de l'ancienne civilization, mais trop peu de gens savent comme faire fonctionner ces bidules.

C'est ici que tu fais ton entré, tu es le magicien informatique de ta tribue de nomade. Le chef Bram de la tribu a besoin d'une application de "to do" pour faire la gestion des tâches qui permettront de reconstruction l'humanite. Il fait appel a tes pouvoirs pour lui créer une telle application.

Tu es un peu inquiet de ne pas réussir à effectuer la tâche qui est demandé de toi, mais pas de souci, je serai la pour t'aider dans ton apprentissage de la sorcellerie informatique.

Unix

Tous les ordinateurs Windows ont été détruits par l'apocalypse. De toute facon, Windows 9 était un flop total.

En tant que magicien de l'informatique, tu te veras confronté à la console Unix. Il serait bien de savoir un peu comment ça fonctionne. Pour l'instant, tu peux te contenter de savoir que la commande ls te permet de lister les fichiers dans le repertoire courant et que la commande cd te permet de changer de repertoire courant (change directory). Mais on est en 2016 quand même, tu peux activer le "panel" "Terminal" de l'explorateur de fichier Dolphin si tu es en train d'utiliser Fedora KDE. Ainsi, tu pourras naviguer dans tes répertoires avec un beau GUI et éxecuter les commandes avec ce Panel.

Checklist

Afin de compléter notre tâche, nous allons utiliser Git et Play. Git est un des meilleurs système de gestion de revision. C'est un program très commun et il est probablement déjà installé sur le poste que tu utilises. Execute la commande:

$ git --version

pour savoir si le logiciel est présent sur la machine. Cette commande devrait imprimer la version de Git qui est installé sur le poste, si le terminal dit que la commande Git n'est pas trouvé, alors tu devras procéder à l'installation de Git. Pour plus d'information sur Git, tu peux visiter le site web officiel.

Maintenant, installons Play. Play est un framework de developpement web. Nous avons choisi de faire notre application de "todo" pour le secours de l'humanité sous la form d'une application web afin que le chef Bram puisse vérifier c'est todo même lorsqu'il est partie en conquête de territoire lointain. En effet, il pourra consulter ces todos sur n'importe quel terminal branché au web magique. Rendez-vous sur le site web www.playframework.org et téléchargez la version 2.1 du logiciel. Allez de l'avant et extrayez l'archive et placez le dossier a quelque part dans votre répertoire personnel.

De simple debut

Choisissez un emplacement adéquat comme ~/Projet et éxecutez la commande:

$ play new todo

Play vous demandera de confirmer le nom de l'application, vous pouvez simplement cliquer sur Entrez pour garder le nom todo. Ensuite choisissez l'option 1 (Create a simple Scala application). Play devrait avoir créé un repertoire dénommé todo dans le répertoire courant. Rentrez dans ce répertoire et éxecutez la commande:

$ play

Ceci devrait avoir commencé la console Play. Éxecutez maintenant la commande:

$ [todo] $ run

afin de démarrer l'application web en debug. Vous pouvez maintenant visiter l'adresse localhost:9000 dans votre navigateur afin de voir a quoi ressemble le site web qui sert d'exemple.
